# Weekly Report Review · 周报初审

对 OKR 周报做第一次初审：先按硬标准打分（3/4/5），再逐条指出问题、给可执行的修改建议。目标是替主管完成第一遍把关，让交到主管手里的周报已经达到 4 分以上。

## 适用场景

- 文档名一般是「OKR 进展报告（姓名 日期）」
- 用户说：初审周报、给周报打分、看看这周周报行不行、本周进展/下周计划写得怎么样

## 评分标准（硬性，按档逐级满足）

打分前先判断：**这份周报 30 秒内能不能让主管看完所有重点？** 看不完，最高 3 分。

### 3 分 —— 合格的格式与结构（基础门槛）
全部满足才到 3 分：
- 符合格式要求，逻辑清晰、结构清晰
- **每段开头有数字**（如 1 / 2 / 3 或 1.1 / 1.2）
- **重点用颜色高亮，且要高亮"对"的地方**（不是通篇黑字，也不是乱标）。明确该高亮的是这三类，可以用不同颜色区分：
  1. 一个项目**最主要的心得与启发**（如"时间规划问题：找不到优先级，接到什么活就做什么活"）
  2. 一个项目的**关键结论或洞察**
  3. **需要解决的问题和预警**
- 某一块如果有 3 个点，**这 3 个点必须各有小标题总结**，不能堆成一段
- **每个 KR 都要有交代**：有进展写进展；本周无进展的 KR，**必须写明预计推进/启动的计划时间点**（哪一周做、为什么这周暂缓）。不允许出现只写"无 / 空"却没有计划时间的 KR——否则不到 3 分。

### 4 分 —— 有反思、有下步方案
在满足 3 分的基础上：
- **不只罗列问题，每个问题都要有下一步方案**：计划怎么做、怎么解决、什么时候解决
- 进展不只写"做了什么"，还要写"结果/效果如何（so what）"
- **下周规划要更详细，每件事有明确的时间节点和 ddl**：精确到"周几什么时候、交付给谁"。例：闪电牛报告周三上午发我、周四上午发给业务方。不能只写"继续推进 X"。

### 5 分 —— 有深度、无 AI 味、重点分明
在满足 4 分的基础上：
- 总结**有深度**，不是流水账
- **没有 AI 味**：不堆高度概括的空话，是真人思考过的表达
- **重点分明，主管 30 秒能看完所有重点**
- 既有反思又有计划，形成闭环

> 低于 3 分：格式/结构没达标、没有数字开头、没有小标题、通篇流水账、重点埋在文字里。

## 高频问题清单（逐项核对，命中即指出）

格式与结构：
- [ ] 流水账、没有分级（点状项目新增 vs 点线面汇总应分开讲）
- [ ] 文字多却没有小标题
- [ ] 段落没有以数字/结论开头
- [ ] 重点没有用颜色高亮，或高亮错地方（该高亮的是：心得启发 / 关键结论洞察 / 待解决问题与预警）

内容与逻辑：
- [ ] 进展没讲清状态：已完成什么 / 进行中什么 / 当前进度 / 何时完成
- [ ] **有 KR 写"无 / 空"却没写预计推进时间**（双月复盘会变成"被其他项目搁置"的甩锅口，必须当周就锁定计划时间）
- [ ] 只罗列问题，没写解决方案
- [ ] 写了动作/工具尝试，但没有结论（so what：要换掉还是保留？能用不？效果如何？）
- [ ] 信息不闭环：提到的项目没头没尾（"什么项目""之前那个怎么样了""谁验收"）
- [ ] 没写下周工作重点 / 多人协作时没写分工
- [ ] **下周规划不够细：没有明确时间节点和 ddl**（要精确到"周几什么时候交付给谁"，不能只写"继续推进"）
- [ ] 计划不留 buffer、卡点交付（没给方案沟通和多轮修改留时间）

表达：
- [ ] 语病、标点错误、"不是一个句子是一个名词"这类问题

## 审阅口径

- **先打分，再说问题**。分数要对照上面的档位逐级判断，并说明"卡在哪一档、差什么才能上一档"。
- 问题按严重程度排序：先结构/逻辑/闭环，再表达/标点。
- 每条问题要**引用周报原文位置**，并给出**能直接照着改**的建议，不要泛泛而谈。
- 看到"做了 X"但没有结果时，必须追问 so what。
- 看到罗列问题时，必须追问解决方案和时间。
- 看到计划时，检查是否留 buffer、是否卡最后一天交付。
- **看到任何 KR 写"无 / 空 / 暂无进展"时，必须要求补上"预计哪一周推进/启动"**。理由直接说明：双月结束复盘时，没有计划时间的 KR 会变成"被其他项目搁置"的扯皮点；当周就写清计划时间，才能在双月里被追踪。

## 输出格式

```markdown
## 评分：X / 5
[一句话说明卡在哪一档、差什么能上一档]

## 30 秒重点还原
[尝试只读高亮和小标题，复述这周重点。如果还原不出来，说明重点不分明]

## 主要问题（按严重度排序）
### 1. [问题类型]
- 原文位置：[引用周报里的标题/句子]
- 问题：[为什么不行]
- 改成：[可直接套用的改法]

### 2. ...

## 下一步修改清单
- [ ] ...
- [ ] ...
```

## 收尾鼓励（每次审核完都要给）

审核再严，最后也要给一句真诚的鼓励——指出问题是为了让周报更好，不是否定人。**每次审核结束，从下面随机抽 1 条**放在最末尾（不要每次都用同一条，也不要一次列好几条）。语气像同事，不要油腻、不要打分式夸。

1. 这周的结构比上次清楚多了，剩下的都是小修，照清单过一遍就行，别有压力。
2. 能看出来你认真复盘了，方向对的，把重点再拎一拎就很顺了。
3. 列问题不是挑刺，是这份值得改到更好——补上计划和方案就闭环了。
4. 你已经踩到关键了，差的只是说得更利落，这比从零写容易多了，加油。
5. 周报是练出来的，你这版的进步是实打实的，继续保持。
6. 有几处反思我挺认可的，把它们顶到前面、加上下一步，整篇就立住了。
7. 写到这程度不容易了，剩下是"术"的问题，多过几遍就成肌肉记忆。
8. 每周都在长进，这就是在变强，辛苦了，先改最关键的两三条就够明显。
9. 标准定得高是因为相信你够得着，改完再发我，我们一起磨成范例。
10. 这周信息量不小，辛苦。重点先收敛，让我 30 秒看完，就成功一大半了。
