# Research Report Review · 研究报告初审

对研究报告做第一次初审，替主管完成第一遍把关并**打分（3/4/5）**。核心信条（主管反复强调）：

> **总结是一份报告最重要的部分；报告要为业务目的服务；要 30 秒看到重点；思考不能外包给 AI。**

## 适用场景

- 行业研究 / 竞品分析报告（如单词营、计算营竞品拆解）
- 市场调研 / 用户调研 / 满意度报告 / 用户画像
- 用户说：初审报告、给这份报告打分、报告结构/总结怎么改

## 打分标准（必须给一个分数：3 / 4 / 5）

评分是**逐级递进**的：先满足 3 分的全部格式门槛，才看 4 分的深度，再看 5 分的精炼。任一 3 分要件不达标，**最高只能给到 2 分以下（不及格，直接打回）**。

### 3 分 —— 格式与结构门槛（缺一不可，达不到直接打回）

1. **每个一级标题最上面都必须有"本部分总结"。** 现在最常见的硬伤就是一级标题下面直接进正文、没有先给总结。**没有就不到 3 分。**
2. **总结必须有数字 + 层级结构**：用 `1.` `2.` 编号，并且有 `1.1` `1.2` 的二级拆分，**至少 2–3 层**。不允许一段大白话铺到底。
3. **30 秒能看到重点**：重要结论要**高亮**（加粗/底色/callout），但
   - 高亮不能太多（满屏高亮 = 没有重点）；
   - **颜色不能超过 2 种**。
4. **报告整体总结同样要符合上面的规则**：有结构（至少 2–3 层）、有数字、重点高亮克制。
5. **必须有"研究背景和目的"（最关键的地基项）**：为什么做这个研究、要解决/支撑什么业务问题或决策、给谁用。
   - 这是后面所有判断的前提：**只有先看到研究背景和目的，才能判断报告的重点是否契合目的、议题拆解是否相关、是否"准"。**
   - 因此，**研究背景和目的缺失或含糊（只写了研究方式/样本、没写为什么做和支撑什么决策），即使其它都不错，最多也只能给 3 分**——因为无法验证 4 分要求的"准"。
6. **必须有"研究议题拆解"**：把研究目的拆成几个要回答的问题/板块（即目录式的议题地图），让人知道报告要回答哪几件事。议题必须从研究目的推导而来。

### 4 分 —— 先"准"再"深"（在 3 分基础上，顺序不能反）

判 4 分有先后：**先看准不准，再看有没有深度。不准就别谈深度。**

1. **准（关联度，先决条件）**：在研究背景和目的明确的前提下——
   - 每个部分的结论、整体结论的重点，是否**契合研究背景和目的**；
   - 是否**和研究议题相关或是其合理延伸**，有没有跑题；
   - 每个洞察是否回扣"这对业务/对洋葱意味着什么"（参考范例的"对比洋葱的启发""结论："）。
   - ⚠️ 研究背景目的不清时，"准"无从判断，**直接卡在 3 分**，不进入下一步。
2. **深（深度，在准的基础上）**：每个部分的总结是否**概括了该一级标题下的所有重点**——做到"只看这段总结，就不用看下面正文"。总结漏掉正文关键发现 = 深度不够。

### 5 分 —— 通俗、精炼、重点突出（在 4 分基础上）

1. 整体**通俗易懂**，外行也能读懂，不堆术语。
2. **尽可能精炼、缩短文字数量**：能用图/表/一句话说清的，绝不写一大段。
3. **重点足够突出**，主管扫一眼就抓到核心。
4. **需要了解的背景/支撑信息，用引号（引用块 blockquote）隐藏**——正文只留结论，证据、原话、数据细节折叠进引用块。这是范例报告最显著的手法。

## 对标范例（5 分长什么样）

- 《竞品扫描核心洞察结论》：标题即结论；开篇 `0️⃣研究背景和方式`（外因/内因 → 研究目的：提高销售转化、为课程团队总结卖点 → 研究方式&人员 → 竞品选择逻辑）；`1️⃣核心洞察和结论`/`2️⃣关键小总结` 用数字编号、`3.1/3.2/3.3` 分层；每个洞察后跟"对比洋葱的启发/结论："；**大量证据、达人原话、销售话术全部塞进引用块**，正文只留结论；callout 克制。
- 《高中用户画像调研报告》：标题 `🔴 主要结论速读`，callout"一分钟报告精华速读"；每条结论一行话 + 引用块隐藏支撑（如 `> 8成压力大→时间少任务重→近7成不会安排时间`）；明确"研究方法&样本""项目背景与概览（面临的业务问题）"；给阅读指引"完整报告 1h+，时间紧可只看每部分小结和五六部分"；每部分有 🌈小结；结尾 Who/What/How 总结。

> 判分时的对照动作：把被审报告和这两份范例并排比——范例的开头、每部分总结、引用块用法、高亮密度，被审报告差在哪一项，就是扣分点。

## 审阅口径

- **先看每个一级标题有没有总结、有没有研究背景/目的/议题拆解**——这是 3 分硬门槛，缺了直接打回，不用往下看深度。
- **研究背景和目的是地基，不能含糊。** 没有明确的研究背景和目的，就**无法判断重点是否契合目的、议题是否相关，也就无法判断"准"**——这种报告即便结构和数据都不错，**最多 3 分，不能进 4 分**。
- **判 4 分的顺序是"先准再深"，不能反。** 先确认契合背景/目的、议题相关（准），再看总结能否替代正文（深）。不准就停在 3 分。
- **总结写得不行，整份报告先打回**——这是主管最看重的。
- 判断"深度"用一个测试：**只读总结能不能替代读正文**？不能就是 4 分不达标。
- 看到深度分析某个竞品，必须追问"为什么是这个品、想看什么、和研究目的/业务什么关系"。
- 看到一屏好几种颜色/满屏高亮，直接扣 3 分项（颜色≤2 种、高亮克制）。
- 文字密、背景信息和正文混在一起时，要求把背景/原话/数据折叠进引用块（5 分手法）。
- 提醒：先自己思考再用 AI 辅助，不要让 AI 代替思考（AI 味、空话、缺活人感要点出来）。

## 输出格式

```markdown
## 评分：X / 5
[1-2 句：为什么是这个分，卡在下一档的关键原因]

## 3 分门槛核对（逐项，缺一即打回）
- [ ] 每个一级标题最上面都有总结
- [ ] 总结有数字 + 2–3 层结构（1. / 1.1）
- [ ] 重点有高亮、颜色 ≤2 种、不过量
- [ ] 整体总结同样合规（有结构有数字）
- [ ] 有研究背景和目的
- [ ] 有研究议题拆解

## 4 分（深度 + 关联度）
- 深度：只看总结能否替代正文 —— [说明，指出漏了哪些重点]
- 关联度：各部分/整体结论是否契合研究背景与目的、是否回扣业务 —— [说明]

## 5 分（精炼 + 突出 + 背景折叠）
- 是否通俗精炼、重点突出、背景信息折叠进引用块 —— [说明]

## 主要问题（按严重度排序）
### 1. [问题类型]
- 原文位置：[引用标题/句子/图表]
- 问题：[为什么不行 / 会导致什么业务判断风险]
- 改成：[可执行的改法、要补的数据或结论]

## 可保留的部分
- [清楚、有价值、可沿用的内容]

## 下一步修改清单
- [ ] ...
```

## 收尾鼓励（每次审核完都要给）

审核再严，最后也要给一句真诚的鼓励——指出问题是为了让报告更好，不是否定人。**每次审核结束，从下面随机抽 1 条**放在最末尾（不要每次都用同一条，也不要一次列好几条）。语气像同事，不要油腻、不要打分式夸。

1. 这版的骨架比上次清楚多了，剩下的都是打磨，别有压力，照着清单改一遍就行。
2. 能看出来你这次在结构上下了功夫，方向是对的，咱们再抠几个细节就很能打了。
3. 问题都列出来不是挑刺，是因为这份底子值得被改到更好——改完肯定能上一个台阶。
4. 你已经踩到关键点了，差的只是把它说得更利落，这个比从零开始容易多了，加油。
5. 别被一堆批注吓到，研究报告本来就是改出来的，你这版进步是实打实的。
6. 这份里有几处洞察我挺喜欢的，把总结再收一收，整份报告的价值就立起来了。
7. 写到这个程度已经不容易了，剩下的都是"术"的问题，多过几遍手就成肌肉记忆了。
8. 改报告最磨人，但你每改一版我都能看到长进，这就是在变强，继续。
9. 标准定得高是因为相信你够得着，按清单改完再发我，我们一起把它磨成范例。
10. 辛苦了，这份的工作量不小。问题集中改 2-3 个最关键的就够明显了，不用一次到位。
