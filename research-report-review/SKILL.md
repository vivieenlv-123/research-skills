# Research Report Review · 研究报告初审

对研究报告做第一次初审，替主管完成第一遍把关并**打分（3/4/5）**。核心信条（主管反复强调）：

> **总结是一份报告最重要的部分；报告要为业务目的服务；要 30 秒看到重点；思考不能外包给 AI。**

## 适用场景

- 行业研究 / 竞品分析报告（如单词营、计算营竞品拆解）
- 市场调研 / 用户调研 / 满意度报告 / 用户画像
- 用户说：初审报告、给这份报告打分、报告结构/总结怎么改

## 打分标准（必须给一个分数：3 / 4 / 5）

评分是**逐级递进**的：先满足 3 分的全部格式门槛，才看 4 分的深度，再看 5 分的精炼。任一 3 分要件不达标，**最高只能给到 2 分以下（不及格，直接打回）**。

> **核心原则：3 分不是"有这些要素"就行，要素还必须"对"。** 一份报告的源头是「研究背景和目的」——如果它写得不准，会连环传导：
> - 研究背景目的不准 → **研究议题就有无数种可能，根本没法评估议题准不准**（议题准的前提是"和调研目的相关、能回答业务方的问题"）；
> - 议题不准 → **报告结论整体是歪的，不一定是真正的重点**；
> - 结论歪 → **"对业务/对洋葱的启示"也无从呼应**（启示必须和结论呼应、紧扣调研背景）。
>
> 所以审report先卡死源头：**研究背景和目的写得不对，整份报告卡在 3 分以下，不进 4 分**——因为后面的"准"全都无法成立。

### 3 分 —— 格式与结构门槛（缺一不可，达不到直接打回）

1. **每个一级标题最上面都必须有"本部分总结"。** 现在最常见的硬伤就是一级标题下面直接进正文、没有先给总结。**没有就不到 3 分。**
2. **总结必须有数字 + 层级结构**：用 `1.` `2.` 编号，并且有 `1.1` `1.2` 的二级拆分，**至少 2–3 层**。不允许一段大白话铺到底。
3. **30 秒能看到重点**：重要结论要**高亮**（加粗/底色/callout），但
   - 高亮不能太多（满屏高亮 = 没有重点）；
   - **颜色不能超过 2 种**。
4. **报告整体总结同样要符合上面的规则**：有结构（至少 2–3 层）、有数字、重点高亮克制。
5. **必须有"研究背景和目的"，而且要"对"（最关键的地基项）**。光有这个标题、有几句话还不够，内容必须是**这个项目调研本身的背景和目的**，不是别的。判断它对不对，看有没有讲清这几件事：
   - **业务方是谁**：要出现具体的业务部门——课程、产品、市场策略等——是谁的需求；
   - **遇到了什么业务问题**：业务方当前卡在哪、痛在哪，所以才发起这次调研；
   - **为什么做这个调研、希望达到什么目的**：这次调研要回答业务方的什么问题；
   - **怎么应用**：结论拿去支撑哪个决策/动作。
   - ⚠️ 常见错位 1：把"**为什么选这个竞品来研究**（比如它和我们形态相似）"写成了研究背景和目的。这是"选品理由"，**不是"我们为什么做这个项目调研"**——标题和内容对不上，判为"不对"。
   - ⚠️ 常见错位 2：把"**研究内容**"当成研究目的。"拆解 XX 在获客/转化/课程设计上的做法"是研究内容（要做什么），不是目的（为业务方解决什么问题）。
   - **背景目的缺失、含糊、或写得不对（错位成选品理由、没有业务方/业务问题/应用），整份报告最多 3 分**，不进 4 分。
6. **必须有"研究议题拆解"，且议题要"准"**：把研究目的拆成几个要回答的问题/板块（目录式议题地图）。
   - 议题准的前提是 **和调研目的相关、能回答业务方的问题**。
   - ⚠️ 如果上面的背景目的没写清/不对，**议题就有无数种可能，根本无法评估哪个准** —— 这种情况议题再工整也不算达标。

### 4 分 —— 先"准"再"深"（在 3 分基础上，顺序不能反）

判 4 分有先后：**先看准不准，再看有没有深度。不准就别谈深度。**

1. **准（关联度，先决条件）**：在研究背景和目的**写得对**的前提下——
   - 每个部分的结论、整体结论的重点，是否**契合研究背景和目的**、是否是业务方真正要的重点；
   - 是否**和研究议题相关或是其合理延伸**，有没有跑题；
   - 每个洞察/启示是否**和上面的结论呼应、紧扣调研背景**——"对业务/对洋葱的启示"不能是另起炉灶、和结论没关系的一段（启示必须由结论生长出来）。
   - ⚠️ 研究背景目的不清**或不对**时，"准"无从判断，**直接卡在 3 分**，不进入下一步。
2. **深（深度，在准的基础上）**：每个部分的总结是否**概括了该一级标题下的所有重点**——做到"只看这段总结，就不用看下面正文"。总结漏掉正文关键发现 = 深度不够。

### 5 分 —— 通俗、精炼、重点突出（在 4 分基础上）

1. 整体**通俗易懂**，外行也能读懂，不堆术语。
2. **尽可能精炼、缩短文字数量**：能用图/表/一句话说清的，绝不写一大段。
3. **重点足够突出**，主管扫一眼就抓到核心。
4. **需要了解的背景/支撑信息，用引号（引用块 blockquote）隐藏**——正文只留结论，证据、原话、数据细节折叠进引用块。这是范例报告最显著的手法。

## 高频问题清单（主管在真实报告上反复批的点，逐条核对）

这些是从主管批注里提炼的高频问题，审报告时逐条过一遍，命中就指出。

**一、目的/内容/选品理由，三者别混**
- [ ] 把"研究内容"当成"研究目的"——"拆解 XX 在获客/转化/课程设计上的做法"是**研究内容**，不是目的；目的是"为业务方解决什么问题"。
- [ ] 把"为什么选这个竞品研究"当成研究目的（地基项，见 3 分门槛）。
- [ ] 选了某个品做深度拆解，却没讲清"**为什么拆它、具体想看什么**"。
- 主管原话："这是研究内容不是目的""深度分析这个品的原因是什么？拆解这个品具体想看什么？"

**二、结论 / 启示 / 全文必须互相呼应、内部一致**
- [ ] **启示不是从结论里长出来的**：对业务/对洋葱的启示是另起炉灶的新内容，和上面的结论没关系。启示必须呼应结论、紧扣调研背景。
- [ ] **整体总结和下面各部分小总结对不上**：要逐一对照每部分小结，确认总结是从正文长出来的。
- [ ] **报告内部结论自相矛盾**：开头结论说"增长靠品牌×低价"，后文却说"最大差异化是书写词汇+听写"；或结论说"没对应在校阶段"但自己的图里明明有学段对应。
- [ ] **漏掉业务方最关心的点**：如课程侧关注的版本同步、全流程 SOP、分班机制等没写进结论。
- 主管原话："启发和上面的结论有相关性吗""你的整体总结和启发没关系""这里和后面的结论不太一致""课程侧会最关注什么"。

**三、自己发现的问题，要自己给出答案/归因/猜测**
- [ ] 只抛问题、只下判断，不给"为什么"和"那又怎样"。如写"承诺兑现失败"却没说为什么会失败；写"转化承接不足"却不深挖原因。
- [ ] 发现重要结论（如竞品续费没跑通）不顺势追问"为什么？我们是否也会跑不通？那还要不要做？"
- 主管原话："自己发现的问题，需要自己给出答案或者猜测""需要简单补充为什么兑现失败""你需要思考……为什么呢？我们是不是也有可能跑不通"。

**四、结论里的名词/概念要解释清楚，主动消除读者疑惑**
- [ ] 结论或正文抛出专业名词/自造概念却不解释：如"六步闭环（哪六步？）""线性词频""轻陪跑（怎么个轻）""未拆封（有硬件吗）"。
- [ ] 尤其写在最前面的结论里的概念，读者还没看后文，更要当场说清，并说明"为什么成立"（如"为什么有信服力"）。
- 主管原话："哪六步？写在最开头的结论里……就要尽量消除疑惑""什么是线性词频""具体的轻陪跑是什么"。

**五、结论与证据要一致、且证据要够**
- [ ] **结论和证据打架**：说"口碑差异化/亮点"，但后面用户体验全是吐槽，没有好评佐证。
- [ ] **单一来源/单一渠道就下大结论**：如只看小程序销量就判"转化不足"，没看达人直播间等其他渠道；服务力度结论只来自小红书评论、没联系销售验证。
- 主管原话："是宣传差异化还是真口碑？如果是口碑需要给出好评的点""除了小程序是否还有其他销量渠道""证据不足"。

**六、内容归类要准确、放对章节**
- [ ] 归类错位：把"达人测评矩阵 + App 内购"（营销渠道）塞进"产品定位"。
- 主管原话："这块和卖点部分更接近，营销渠道和方式，不算是产品定位"。

**七、研究爆款/竞品 top1，先答"它为什么卖得好"**
- [ ] 把研究对象的亮点写成了槽点；研究销量第一的品，要先回答"它**为什么是冠军**（尤其没护城河还卖得好）"，以学习为主，再谈它的问题。
- 主管原话："这是销量最好的，是有原因的吧""先解决为何他是冠军，尤其是没护城河还能卖得好""谁说亮点一定要有护城河"。

## 对标范例（5 分长什么样）

- 《竞品扫描核心洞察结论》：标题即结论；开篇 `0️⃣研究背景和方式`（外因/内因 → 研究目的：提高销售转化、为课程团队总结卖点 → 研究方式&人员 → 竞品选择逻辑）；`1️⃣核心洞察和结论`/`2️⃣关键小总结` 用数字编号、`3.1/3.2/3.3` 分层；每个洞察后跟"对比洋葱的启发/结论："；**大量证据、达人原话、销售话术全部塞进引用块**，正文只留结论；callout 克制。
- 《高中用户画像调研报告》：标题 `🔴 主要结论速读`，callout"一分钟报告精华速读"；每条结论一行话 + 引用块隐藏支撑（如 `> 8成压力大→时间少任务重→近7成不会安排时间`）；明确"研究方法&样本""项目背景与概览（面临的业务问题）"；给阅读指引"完整报告 1h+，时间紧可只看每部分小结和五六部分"；每部分有 🌈小结；结尾 Who/What/How 总结。

> 判分时的对照动作：把被审报告和这两份范例并排比——范例的开头、每部分总结、引用块用法、高亮密度，被审报告差在哪一项，就是扣分点。

## 审阅口径

- **先看每个一级标题有没有总结、有没有研究背景/目的/议题拆解**——这是 3 分硬门槛，缺了直接打回，不用往下看深度。
- **研究背景和目的是地基，既要"有"也要"对"。** 检查它是不是"这个项目调研的目的"——有没有业务方（课程/产品/市场策略）、业务问题、调研目的、怎么应用。常见错位是把"为什么选这个竞品研究"当成研究目的。**背景目的缺失、含糊或写得不对，都无法判断"准"——最多 3 分，不能进 4 分。**
- **背景目的不准 → 议题无法评估、结论可能是歪的、启示无从呼应**，这是连环传导，审的时候要顺着这条链一路点出来，不要只夸"有结构"。
- **检查启示/建议是否和结论呼应。** "对业务/对洋葱的启示"必须由上面的结论生长出来、紧扣调研背景；如果它自说自话、和结论没关系，即使内容是业务方关心的，也算没"准"。
- **判 4 分的顺序是"先准再深"，不能反。** 先确认契合背景/目的、议题相关（准），再看总结能否替代正文（深）。不准就停在 3 分。
- **总结写得不行，整份报告先打回**——这是主管最看重的。
- 判断"深度"用一个测试：**只读总结能不能替代读正文**？不能就是 4 分不达标。
- 看到深度分析某个竞品，必须追问"为什么是这个品、想看什么、和研究目的/业务什么关系"。
- 看到一屏好几种颜色/满屏高亮，直接扣 3 分项（颜色≤2 种、高亮克制）。
- 文字密、背景信息和正文混在一起时，要求把背景/原话/数据折叠进引用块（5 分手法）。
- 提醒：先自己思考再用 AI 辅助，不要让 AI 代替思考（AI 味、空话、缺活人感要点出来）。

## 收尾鼓励（每次审核完都要给）

审核再严，最后也要给一句真诚的鼓励——指出问题是为了让报告更好，不是否定人。**每次审核结束，从下面随机抽 1 条**放在最末尾（不要每次都用同一条，也不要一次列好几条）。语气像同事，不要油腻、不要打分式夸。

1. 这版的骨架比上次清楚多了，剩下的都是打磨，别有压力，照着清单改一遍就行。
2. 能看出来你这次在结构上下了功夫，方向是对的，咱们再抠几个细节就很能打了。
3. 问题都列出来不是挑刺，是因为这份底子值得被改到更好——改完肯定能上一个台阶。
4. 你已经踩到关键点了，差的只是把它说得更利落，这个比从零开始容易多了，加油。
5. 别被一堆批注吓到，研究报告本来就是改出来的，你这版进步是实打实的。
6. 这份里有几处洞察我挺喜欢的，把总结再收一收，整份报告的价值就立起来了。
7. 写到这个程度已经不容易了，剩下的都是"术"的问题，多过几遍手就成肌肉记忆了。
8. 改报告最磨人，但你每改一版我都能看到长进，这就是在变强，继续。
9. 标准定得高是因为相信你够得着，按清单改完再发我，我们一起把它磨成范例。
10. 辛苦了，这份的工作量不小。问题集中改 2-3 个最关键的就够明显了，不用一次到位。
