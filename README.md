# research-skills

团队研究标准文档

## 工具

面向用户 / 内容研究的命令行工具，每个子目录可独立安装使用。遵循同一理念：**CLI 做机械活（取数、清洗、存档、写入），AI 做判断活（总结、分类、洞察）**。

| 子目录 | 说明 |
|--------|------|
| [`wechat-mp-cli`](wechat-mp-cli) | 收集、提取、总结、归档微信公众号文章的 CLI，配合 AI 代理使用，可把文章链接与总结沉淀进飞书文档。 |

## 评审 Skill

替主管完成第一遍把关的初审 Skill，按 3/4/5 分标准打分、逐条给可执行的修改建议。把对应文件夹放到项目的 `.agents/skills/` 下即可使用。

| 子目录 | 说明 |
|--------|------|
| [`research-report-review`](research-report-review) | 研究报告（竞品分析 / 调研 / 用户画像 / 满意度）初审。核心标准：每个一级标题都要有总结、必须有研究背景和目的与议题拆解、先"准"再"深"。 |
| [`weekly-report-review`](weekly-report-review) | OKR 周报初审。核心标准：数字开头、重点高亮、有反思有计划、每个 KR 都要交代（无进展也要写预计推进时间）。 |
